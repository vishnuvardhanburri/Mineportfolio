## üîß Want to Submit Code?

### Getting Started

1. **Fork the repository**
2. **Clone your fork**
   ```bash
   git clone https://github.com/YOUR-USERNAME/portfolio.git
   cd portfolio
   ```

3. **Create a branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Install dependencies**
   ```bash
   npm install
   ```

5. **Make your changes**

### Code Guidelines

- **TypeScript**: Use TypeScript with proper typing
- **React**: Follow React best practices and hooks conventions
- **Tailwind CSS**: Use Tailwind utility classes (avoid custom CSS when possible)
- **Formatting**: Code will be formatted automatically, but try to match the existing style
- **Components**: Keep components focused and reusable
- **Naming**: Use clear, descriptive names for variables and functions

### Testing Your Changes

```bash
# Start dev server
npm run dev

# Build for production (check for errors)
npm run build

# Preview production build
npm run preview
```

### Commit Guidelines

Use clear, descriptive commit messages:

```bash
# Good commits
git commit -m "Fix: Portal animation timing on mobile devices"
git commit -m "Feature: Add skill category filtering"
git commit -m "Docs: Update README installation steps"

# Avoid
git commit -m "fixes"
git commit -m "update"
```

### Submit a Pull Request

1. **Push your branch**
   ```bash
   git push origin feature/your-feature-name
   ```

2. **Create a Pull Request** on GitHub with:
   - Clear title describing the change
   - Description of what changed and why
   - Reference to any related issues
   - Screenshots (for UI changes)

3. **Wait for review**
   - I'll review as soon as possible
   - Be open to feedback and suggestions
   - Make requested changes if needed
